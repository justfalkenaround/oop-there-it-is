DO NOT CHEAT

It should go without saying that any of the code examples and assets should not be copied directly for the purpose of academic submissions. That being said you are more than welcome to use the knowledge, assets, and small sections of code for your own projects as long as you leave an attribution. Don't take credit for something you didn't create. Odds are that the truth will come out and it will only lead to suffering for you and the people around you. I have no doubt that if you apply yourself, you will be able to create your own original content that you can take pride in.

NOTE: Finished code examples will be provided with each lesson. Though it may be tempting to peek at the code before completing the lesson challenge, don't do it. Nobody is going to stop you from doing it, but it will diminish the satisfaction and learning you will get if you complete the challenge as intended.

STRICT MODE

Every JAVASCRIPT file in this course will and should have strict mode enabled. As you will see at the very first line of every file exactly. 

'use strict';

Strict mode will throw some errors when JAVASCRIPT would normally not. This is good, it will help prevent insidious little bugs that come from minor syntax errors that might have slipped through the cracks otherwise. However it is not a panacea, you must still me mindful to avoid and catch general syntax and semantic errors.

ASYNC AWAIT

All of the code associated with this course will adopt the newer async await syntax instead of using the old "Promise.then().catch()" syntax. Coding challenges should be completed using this syntax for async control.

BEFORE ASYNC AWAIT

const before_async = () => {
  
  new Promise((resolve, reject) => {
    window.setTimeout(() => resolve(), 3000);
  })
  .then(() => console.log('I WAITED 3 SECONDS'))
  .catch(err => console.error('SOMETHING WENT WRONG', err));
    
};

WITH ASYNC AWAIT

const with_async = async () => {
  
  try {
    await new Promise((resolve, reject) => {
      window.setTimeout(() => resolve(), 3000);
    });
    console.log('I WAITED 3 SECONDS');
  }
  catch(err) {
    console.error('SOMETHING WENT WRONG', err);
  }

};

This syntax may be new to you but it can make complicated asynchronous code much easier to read without changing any functionality. Please read the section in dev-docs related to JAVASCRIPT async await if you are not familiar.

CLASS SYNTAX

All of the code associated with this course will adopt the newer JAVASCRIPT class syntax for the purpose of object oriented programming. We will be using them liberally throughout the course. You will learn about classes as the lessons proceed, however, it is advised that you read up on the basics of the syntax.

CLASS DOCS

You may come across (much) older code that does not utilize the class syntax for it's object oriented approach but rather the old underlying object methods or syntax. This code will still run and is in fact the the same exact functionality that classes use, just harder to learn.

For simplicity, this course will not be teaching this old syntax, however if you wish to gain a deeper understanding of JAVASCRIPT and how it works "under the hood", you might consider becoming familiar with it after completing this course.

BEFORE CLASS

function Vector(x, y) {
    this.x = x;
    this.y = y;
}

Vector.prototype.distance = function(v) {
    return Math.sqrt(Math.pow(this.x - v.x, 2) + Math.pow(this.y - v.y, 2));
}

WITH CLASS

class Vector {
    constructor (x, y) {
        this.x = x;
        this.y = y;
    }

    distance(v) {
        return Math.sqrt(Math.pow(this.x - v.x, 2) + Math.pow(this.y - v.y, 2));
    }
}

I am confident that if you take the time to adopt these syntax features and techniques that you won't go back.
